== Terminology (the thing I got sloppy on)

`pyproject.toml` is the *project-definition manifest* (PEP 621 `[project]` + PEP 518 `[build-system]`), not "a config file."

* *Build tool* (Ant): imperative -- you script the steps, the *how*. No project model.
* *Project-definition tool* (Maven, Gradle): declarative project *model* -- you declare what the project *is*; the build is derived.

In Python: the manifest is `pyproject.toml`. *Build backends* (setuptools, hatchling, flit-core, pdm-backend, uv_build) do the build. *Project/dependency managers* -- the Maven/Gradle analog -- are **Poetry, PDM, Hatch, uv**. (pip, conda, Pipenv = install/env, not project definition.)

== Which project-definition tool

[cols="1,3"]
|===
| Axis | Finding

| Popularity `[V]`
| Survey 2024 (dependency mgmt): Poetry *20%*, uv *12%*; Hatch/PDM not in reported set. uv fastest-growing (11% in year one).

| Maturity `[V]`
| setuptools 2004 > Flit 2015 > Poetry 2018-02-28 > Hatch 1.0 2022-04 > uv 2024-02-15. PDM first-release date *not reliably verified* -- excluded.

| Practicality `[J]`
| uv or Poetry for this repo (IntelliJ IDEA + existing conda `ml` env).

| Capability `[J]`
| uv is broadest: project + deps + lockfile + Python-version + env + build backend; fastest (Rust).

| IntelliJ IDEA -- *not* PyCharm `[V]`
| Native, auto-detected SDK types: *Poetry, uv, Hatch* (+ virtualenv, conda, Pipenv). *PDM is not listed.* setuptools is a backend, not an env type.
|===

*Decision:* **uv** (most capable, IDE-native, fastest-growing) or **Poetry** (most popular/mature, IDE-native). PDM is out -- capable, but IntelliJ IDEA won't drive it.
